# Spotify for Artists
## Modelo de Royalties por Streaming

### Perfil
- **Empresa:** Spotify Technology S.A.
- **Fundación:** 2006, Estocolmo
- **Stats (2025):** 713M MAU, 281M suscriptores de pago
- **Revenue (2024):** €15.67B
- **Modelo:** Streaming con royalties pro-rata

### Modelo de Royalties

#### Distribución de Revenue
| Destino | Porcentaje |
|---------|------------|
| **Rights holders** | ~70% del revenue |
| **Spotify retiene** | ~30% |

**Importante:** El 70% va a "rights holders" (sellos, distribuidores), NO directamente a artistas.

#### Cadena de Pago Real
```
Spotify revenue
└── 70% → Rights holders (sellos/distribuidores)
    └── Variable% → Artistas (según contrato)
        └── Típico: 15-25% para artistas firmados
        └── Indie: hasta 80-100% si distribuyen directo
```

### Sistema Pro-Rata

#### Cómo Funciona
```
Total streams del mes: 1,000,000,000
Tus streams: 1,000,000
Tu share: 0.1%

Pool de royalties: €100,000,000
Tu pago: €100,000 (antes de splits)
```

#### Pay-per-Stream Estimado
- **Promedio global:** $0.003-0.005 por stream
- **Varía por:** País, tipo de cuenta (free vs premium), acuerdos

### Requisitos para Cobrar

#### Umbral Mínimo (2024)
- **1,000 streams** en 12 meses para generar royalties
- Cambio reciente para eliminar fraude/bots
- Tracks con <1,000 streams no generan pago

#### Distribuidores Necesarios
Artistas necesitan distribuidor para subir a Spotify:
- DistroKid (~$20/año)
- TuneCore (~$10-30/año)
- CD Baby (one-time fee)
- Amuse (gratis con comisión)

### Modelo de Negocio Real

#### Artista Indie (100% ownership)
```
10,000 streams
├── Spotify paga: ~$35 (a distribuidor)
├── Distribuidor: ~$3-7 fee
└── Artista recibe: ~$28-32
```

#### Artista con Sello
```
10,000 streams
├── Spotify paga: ~$35 (al sello)
├── Sello se queda: ~$28 (80%)
└── Artista recibe: ~$7 (20%)
```

### Por Qué Este Modelo NO Funciona para Creadores Pequeños

#### Problema del Pro-Rata
- Artistas grandes se llevan la mayoría del pool
- 1M streams = ~$3,500 (antes de splits)
- Imposible vivir de Spotify sin millones de streams
- Favorece hits virales sobre catálogos nicho

#### Críticas Comunes
- Pago demasiado bajo por stream
- Intermediarios se llevan demasiado
- Modelo beneficia a majors sobre indies

### Lecciones para WhatsSound

#### Lo Que NO Copiar
- ❌ Sistema pro-rata (divide pool entre todos)
- ❌ Intermediarios entre plataforma y creador
- ❌ Pagos tan bajos que no motivan
- ❌ Umbrales mínimos altos

#### Lo Que SÍ Aprender
- ✅ Modelo de streaming funciona para discovery
- ✅ Datos y analytics son valiosos para artistas
- ✅ Playlists/curation como herramienta de promoción

### Aplicación Indirecta a WhatsSound

#### Contexto: DJs Reproducen Música de Spotify
Los DJs ya usan música de otros artistas. WhatsSound no compite con Spotify en royalties, pero puede:

1. **Integrar datos de Spotify** para mostrar qué reproduce el DJ
2. **No crear sistema de royalties** — muy complejo
3. **Enfocarse en propinas directas** — mucho más simple

#### Modelo Propinas vs Royalties
| Spotify (Royalties) | WhatsSound (Propinas) |
|---------------------|----------------------|
| Pago por play | Pago por appreciation |
| Céntimos por stream | Euros por propina |
| Indirecto (via sellos) | Directo (fan → DJ) |
| Meses de delay | Instantáneo |

### Propuesta para WhatsSound

| Concepto | Decisión |
|----------|----------|
| Sistema royalties | NO — demasiado complejo |
| Propinas directas | SÍ — simple y efectivo |
| Integración Spotify | Para mostrar tracks (no pagar) |
| Analytics estilo Spotify | SÍ — datos para DJs |

#### Por Qué Propinas > Royalties para DJs
- **Directo:** Fan paga a DJ sin intermediarios
- **Instantáneo:** Dinero el mismo día
- **Justo:** DJ decide su valor, no algoritmo
- **Engagement:** Conexión emocional fan-DJ
