## Framework: Game Thinking

### Definición
Game Thinking es un sistema de 5 pasos para encontrar product/market fit más rápido y construir engagement profundo, basado en cómo los game designers crean experiencias que la gente ama.

### Los 5 Pasos

```
1. Empathize → 2. Envision → 3. Experiment → 4. Evaluate → 5. Evolve
```

## Concepto Central: Mastery Path

### La Evolución del Jugador

```
Newbie → Regular → Enthusiast → Expert/Master
```

Cada etapa requiere diferentes:
- Motivaciones
- Features
- Contenido
- Recompensas

### Diseñar para el Journey Completo

| Etapa | Necesidad | WhatsSound |
|-------|-----------|------------|
| **Newbie** | Onboarding claro, quick wins | Tutorial, primer voto |
| **Regular** | Rutina, hábito | Racha diaria, sesiones recurrentes |
| **Enthusiast** | Profundidad, mastery | Crear sesiones, curar playlists |
| **Expert** | Status, influencia | DJ verificado, mentoreo |

## Core Loop Design

### Estructura de un Core Loop Exitoso

```
      ┌─────────────────┐
      │   TRIGGER       │
      │ (Motivación)    │
      └───────┬─────────┘
              │
              ▼
      ┌─────────────────┐
      │   ACTION        │
      │ (Simple, claro) │
      └───────┬─────────┘
              │
              ▼
      ┌─────────────────┐
      │   REWARD        │
      │ (Inmediata)     │
      └───────┬─────────┘
              │
              ▼
      ┌─────────────────┐
      │   INVESTMENT    │
      │ (Progreso)      │
      └───────┬─────────┘
              │
              └────────────▶ Repeat
```

### Principios del Core Loop

1. **Acción Simple** — Una acción principal, no 10
2. **Feedback Inmediato** — Sin delay en respuesta
3. **Variabilidad** — No siempre el mismo resultado
4. **Progresión** — Cada loop construye sobre el anterior

**WhatsSound Core Loop:**
```
Trigger: "¿Qué música hay ahora?"
Action: Abrir app → Unirse a sesión
Reward: Música en vivo + Reacciones
Investment: Votar → Seguir DJ → Crear perfil
```

## Superfan Development

### El Framework de Superfans

> "No intentes agradar a todos. Encuentra a tus superfans y dales exactamente lo que quieren."

### Características de Superfans
- Usan tu producto obsesivamente
- Lo recomiendan sin que les pagues
- Te dan feedback valioso
- Toleran bugs y friction
- Defienden tu producto

### Cómo Identificarlos
1. **Behavior:** Alta frecuencia de uso
2. **Advocacy:** Hablan de ti públicamente
3. **Contribution:** Crean contenido/valor

**En WhatsSound:**
- DJs que crean sesiones diarias
- Usuarios que invitan amigos
- Contributors de playlists

## Skill Atoms

### Construyendo Habilidades

Un "Skill Atom" es la unidad mínima de aprendizaje/mastery:

```
Action → Feedback → Mastery indicator
```

**Ejemplo WhatsSound:**
- Action: Votar una canción
- Feedback: Ver tu voto reflejado en cola
- Mastery: Badge "100 votos"

### Escalando Skill Atoms

```
Nivel 1: Unirse a sesión
Nivel 2: Votar canción
Nivel 3: Reaccionar
Nivel 4: Seguir DJ
Nivel 5: Pedir canción
Nivel 6: Crear sesión
Nivel 7: Cultivar audiencia
Nivel 8: Monetizar
```

## Product/Market Fit Signals

Amy Jo Kim propone señales específicas:

### Señales Fuertes de PMF
- ✅ Usuarios vuelven sin recordatorios
- ✅ Crecimiento orgánico > paid
- ✅ Usuarios describen el producto con emoción
- ✅ Retention curve se aplana (no cae a cero)
- ✅ Superfans emergen naturalmente

### Señales de Alarma
- ❌ Solo crecimiento con incentivos
- ❌ Alta churn post-onboarding
- ❌ Usuarios no pueden explicar el valor
- ❌ Sin word-of-mouth
